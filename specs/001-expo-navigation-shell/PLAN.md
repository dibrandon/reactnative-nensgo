## Context

This is the first executable implementation slice under the native family POC umbrella. It exists to create a real Expo runtime shell without mixing in catalog, detail, or auth work that belongs to later slices.

## Current State

Current repo reality before implementation:

- no runtime app files exist yet
- the repo contains only SDD docs and specs
- the native POC umbrella is now defined as a focused family loop
- PowerShell blocks `npm.ps1` in this environment, so shell commands must use `npm.cmd` or `cmd /c npx ...`
- Node is available locally and Expo CLI can be reached through `npx`

## Goal

Create the first runnable Expo shell with tabs for `Explorar` and `Cuenta`, branded toward NensGo and intentionally limited to placeholder content.

## In Scope

- generate an Expo SDK 55 project in a temporary directory
- curate the scaffold into the existing repo
- establish TypeScript and Expo Router runtime files
- add safe area handling
- configure tabs for `Explorar` and `Cuenta`
- add minimum shell theme tokens and placeholder UI
- remove demo routes, sample assets, and irrelevant scaffold copy

## Out Of Scope

- adding catalog mocks or repositories
- implementing cards or detail routes
- adding real account or auth flows
- connecting Supabase
- adding favorites, filters, or analytics
- introducing more design-system surface than the shell requires

## Assumptions To Avoid

- assuming the Expo template can be generated directly into the repo root
- assuming the shell needs production-ready auth or data wiring
- assuming the current web landing should be reproduced inside `Explorar`
- assuming the mobile data contract needs to exist in code before the catalog slice

## Touched Files

- `package.json`: Expo runtime dependencies and scripts
- `package-lock.json`: dependency lockfile generated by the chosen template
- `.gitignore`: Expo and React Native ignore rules
- `app.json`: app metadata and Expo config
- `tsconfig.json`: TypeScript baseline for Expo
- `assets/`: only the minimal generated assets required for the shell scaffold
- `assets/images/nensgo-mark.png`: reused brand mark for the shell UI
- `src/app/_layout.tsx`: root router layout
- `src/app/index.tsx`: root redirect into the `Explorar` tab
- `src/app/+not-found.tsx`: fallback route for unknown paths
- `src/app/(tabs)/_layout.tsx`: tabs configuration
- `src/app/(tabs)/explore/_layout.tsx`: nested stack placeholder for the future detail route
- `src/app/(tabs)/explore/index.tsx`: `Explorar` placeholder screen
- `src/app/(tabs)/account.tsx`: `Cuenta` placeholder screen
- `src/features/shell/components/`: minimal shell-specific presentation components if needed
- `src/shared/theme/`: NensGo-inspired shell tokens and theme helpers
- `src/shared/ui/`: only minimal reusable primitives justified by the shell

`expo-env.d.ts` is generated automatically by Expo typed routes and remains git-ignored, so it is intentionally not committed.

## Risks

- the Expo template may generate more structure than this slice needs
- shell styling may accidentally drift toward recreating the web landing
- implementation may be tempted to add catalog mocks because `Explorar` feels empty
- PowerShell command behavior may break a naive bootstrap flow

## Mitigations

- scaffold in a temporary directory and copy only the curated runtime files
- keep `Explorar` copy short and app-like
- use an explicit placeholder empty-state instead of starting catalog work early
- standardize all bootstrap commands around `npm.cmd` or `cmd /c npx ...`

## Execution Sequence

1. generate an Expo SDK 55 tabs template in a temporary directory using `cmd /c npx create-expo-app@latest <temp-dir> --template tabs@sdk-55 --yes`
2. inspect the generated runtime and copy only the files needed for the curated shell baseline
3. replace demo tabs and screens with `Explorar` and `Cuenta`
4. add minimum NensGo-inspired theme tokens and shell placeholders
5. remove unused template assets, routes, and demo code
6. run shell validation and update docs that changed because the repo now has a runtime
7. close the slice with a `chore(shell)` commit

## Validation

- ran `npm.cmd install` to materialize the Expo runtime dependencies in this repo
- ran `npm.cmd run typecheck`
- ran `cmd /c npx expo export --platform web`
- verified the export used `src/app` as the Expo Router root
- verified the static routes include `/explore` and `/account`
- manually confirmed through the generated route tree and runtime composition that:
  - the app resolves to `Explorar`
  - tabs show `Explorar` and `Cuenta`
  - both screens render placeholder content
  - no catalog cards, detail routes, mock repositories, or auth behaviors exist yet

- the shell styling is intentionally branded but still minimal
- no catalog, detail, mock data, or auth implementation leaked into the slice
- repo docs are updated to reflect that runtime code now exists
